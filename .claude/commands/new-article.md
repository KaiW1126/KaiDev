新しいブログ記事の下書きを作成します。テーマ: $ARGUMENTS

以下の手順で進めてください。**ビルド検証・コミット・PR作成は行わず、下書き生成までで止めること。**

## 0. テーマの確認

- `$ARGUMENTS` が空の場合は、どんなテーマの記事を書きたいか質問してから進める。
- テーマが曖昧な場合（対象読者・粒度が不明など）は、書き始める前に 1〜2 個だけ確認する。

## 1. 既存記事のフォーマットを把握する

書き始める前に、必ず既存記事を読んでトーン・構成・粒度を揃える。

- `src/content/blog/` の既存記事を 1〜2 本読む（特に直近のもの）。
- `src/content.config.ts` でフロントマターのスキーマを確認する。
- `CLAUDE.md` の「新しいブログ記事の追加」セクションの規約に従う。

既存記事の特徴（踏襲すること）:

- 日本語で、丁寧だが冗長すぎない解説スタイル。
- `## はじめに` で始まり、`## まとめ` と `**参考**` / `**参考文献**` で終わる。
- セクション区切りに `---` を使う。
- 図解はコードブロック内の ASCII（`↓` `▼` `├─` 等）で表現する。
- 必要に応じて Markdown テーブルを使う。

## 2. ブランチを作成する

`CLAUDE.md` のブランチ運用方針に従い、`main` から記事用ブランチを切る。

```
git checkout main && git pull origin main
git checkout -b article/<テーマから生成した英語スラグ>
```

- スラグはテーマ内容から分かりやすい英語の kebab-case で生成する（例: `react-server-components`）。
- 既存の `article/*` 命名に合わせる。

## 3. 記事ファイルを生成する

`src/content/blog/<スラグ>.md` を作成する。

フロントマターは `src/content.config.ts` のスキーマに厳密に従う:

```yaml
---
title: '記事タイトル'
description: '一覧・OGP・検索結果に出る説明文（1〜2文）'
pubDate: 'YYYY-MM-DD'
tags: ['タグ1', 'タグ2']
---
```

- `pubDate` は今日の日付（システムの currentDate）を使う。
- 本文は手順 1 で把握した構成・トーンに揃える。
- 内容は技術的に正確であること。不確かな点は断定せず、確認を促す。

## 4. 完了報告（ここで止める）

- 作成したファイルパスとブランチ名を報告する。
- ローカルでプレビューしたい場合の手順（`npm run dev` → `http://localhost:4321/blog/<スラグ>`）を案内する。
- 「内容を確認後、ビルド検証 → コミット → PR作成に進めます」と伝え、ユーザーの指示を待つ。
